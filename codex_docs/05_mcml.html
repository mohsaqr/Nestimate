Purpose

MCML groups nodes into higher-level clusters. The result has a macro layer and within-cluster layers.

Use MCML when the node set has meaningful groups, such as cognitive, metacognitive, and social regulation actions.

Build MCML From Event Data

Use build_mcml() when starting from raw sequences or long event logs.

mc <- build_mcml(
  group_regulation_long,
  clusters = clusters,
  method = "sum",
  type = "tna",
  actor = "Actor",
  time = "Time",
  action = "Action"
)

summary(mc)

Counts Instead of Probabilities

Matrix Aggregation With cluster_summary()

Use cluster_summary() when a network matrix already exists. This path is aggregation-only. It does not reconstruct a Markov process from raw sequences.

net_counts <- build_network(
  group_regulation_long,
  method = "frequency",
  action = "Action",
  actor = "Actor",
  time = "Time"
)

mc_from_matrix <- cluster_summary(
  net_counts,
  clusters = clusters,
  method = "sum"
)

mc_from_matrix

#> Cognition          4698          3456   2778

For matrix input, method controls arithmetic aggregation. Use sum for counts, mean for already-normalized matrices, and robust summaries such as median only when they match the question.

Convert Layers to netobjects

as_tna() returns a netobject_group. The first element is macro; the rest are within-cluster networks.

mc_nets <- as_tna(mc)
mc_nets

#> 
#>   Group          Nodes  Edges  Weights

#>   Cognition      3      8      [0.027, 0.881]

The macro and within-cluster views answer different questions. The macro layer describes movement among clusters; the within layers describe movement among states inside each cluster.

Labels

Use labels when node codes need readable names. The labels affect within-cluster node display while macro labels remain cluster names.

label_map <- c(
  discuss = "Discussion",
  synthesis = "Synthesis",
  consensus = "Consensus",
  plan = "Planning",
  monitor = "Monitoring",
  adapt = "Adapting",
  coregulate = "Co-regulation",
  cohesion = "Cohesion",
  emotion = "Emotion"
)

mc_labeled <- build_mcml(
  group_regulation_long,
  clusters = clusters,
  method = "sum",
  type = "frequency",
  actor = "Actor",
  time = "Time",
  action = "Action",
  labels = label_map
)

state_distribution(mc_labeled)

Interpretation Checklist

When reading MCML output, separate four claims:

  1. Which clusters are large in the state vocabulary?
  2. Which clusters receive or send the most macro flow?
  3. Which macro transitions are stable enough to interpret?
  4. Which within-cluster transitions support a finer-grained claim?

Do not read macro edges as if they were node-level edges. They are aggregated cluster-level transitions.
